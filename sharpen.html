Path A

Direct PR

Standard GitHub flow. Cut a topic branch off main (verbs: add/, fix/, sharpen/, evolve/, remove/), commit the change, push, open a PR against main. Maintainer reviews and merges. Best for deliberate fixes, new skills, or anything where you want the change reviewed as one coherent diff.

git checkout -b add/<skill-name> # edit, commit git push -u origin add/<skill-name> gh pr create --base main --fill
Path B

Organic + cron-curated

Live on a long-lived users/<handle> branch. Edit skills as you use them, commit when you like, push when convenient. A nightly cron walks every branch, audits each contribution (regex layer-1 forbidden-pattern scan plus layer-2 Claude semantic check), and synthesises a best-of-breed evolved SKILL.md onto nightly. Maintainer PRs nightly → main. Best for organic evolution where interrupting your work to open a PR per tweak is friction.

git checkout users/<handle> # edit, commit, edit, commit git push origin users/<handle> # cron picks it up nightly

Both paths converge at main. The forbidden-pattern scan runs on the cron path; for direct PRs the equivalent safety check happens in maintainer review. Use whichever flow fits the change.

Runbook → evolve-skill/SKILL.md  ·  Forbidden patterns → tools/forbidden-patterns.txt  ·  Contributing → CONTRIBUTING.md
