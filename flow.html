Shipping into a green main

Many worktrees, many agents — one always-green main.

Every contributor — human or Claude Code agent — works in its own worktree and opens a small PR. Commits pool into the presubmit bay (build, tests, lint, forbidden-pattern scan); a gate meters them into AI review, which batches a cluster into one merged change and drops it into the merge queue. When the queue is full, a human opens the gate — each merge re-tested against the latest main — and main rises, always green, for the next worktree to branch from. Big changes open a draft Plan-Request first, so the owner approves the intent before any code is written.
