Shipping into a green main

Many worktrees, many agents — one always-green main.

Every contributor — human or Claude Code agent — works in its own worktree, never the shared checkout, and opens a small PR. Each PR runs a gauntlet of gates: an automatic presubmit (build, tests, lint, forbidden-pattern scan), then a required AI review, then a human owner LGTM. Survivors enter a merge queue that re-tests each one against the latest main before it lands — so main is always green, and the next worktree branches off something a little richer. Big changes open a draft Plan-Request first, so the owner approves the intent before any code is generated.
